# AGENTS.md

## 最優先（Language）

- ユーザーとの応答は **必ず日本語（です/ます調）** で行う。
- まず結論を述べる。
- 不要な前置き・過剰な謝罪は行わない。
- 出力は簡潔に。必要十分な情報のみ提示する。

## 動作コンテキスト
- 環境: Ubuntu on WSL2
- シェル: bash
- エディタ: VSCode
- パッケージマネージャー: pnpm（corepack 経由）
- 検索ツール: 文字列検索は `rg`（ripgrep）、ファイル検索は `fd`（Ubuntu で `fd` がない場合は `fdfind`）を使う。
- 生成物や重いディレクトリは探索対象から除外する:
  - node_modules, dist, .next, coverage, .turbo, .cache

## ツールとコマンド

- 文字列検索は `rg` を優先する。複数の読み取りは可能な限り並列化する。
- 変更はパッチ方式で行い、差分を最小化する。破壊的操作（例: `rm`、`git reset`）は明示依頼がない限り行わない。
- 実行/テストなど時間のかかるコマンドは、目的を1行で添えてから実行する。承認が必要な場合はその旨を明確にする。
- 可能なら実行する。できない場合は “実行不可の理由” と “ユーザーがやる最小手順” を提示する。

## 出力フォーマット

- コマンド、パス、識別子はインラインコード（`` `...` ``）で示す。
- 最終報告は短い箇条書き中心（目安: 最大6項目）。必要なら「次の一手」を1つだけ提案する。

## コード変更の原則

- 既存の規約・命名・構造を尊重する。
- 差分は最小化する。
- 不要な自動整形やリネームを行わない。
- 表面的な回避ではなく根本原因を修正する。
- 依頼と無関係な不具合は修正せず、必要なら報告のみ行う。
