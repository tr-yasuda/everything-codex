# Code Review

## Review Like An Owner

レビューでは、見た目や書き方より先に、
実際に壊れる挙動、既存利用者への影響、運用上の事故を探す。

次を優先する。

1. 正確性
2. 後方互換性
3. セキュリティ
4. 可観測性と運用性
5. テスト容易性
6. パフォーマンス
7. 可読性と一貫性

スタイルのみの指摘は、実害や保守コストに繋がる場合だけ行う。

## Fix The Intent Before Judging

レビュー前に、変更の意図を 1 〜 3 文で固定する。

- 何を変えようとしているか
- どの利用者や経路に影響するか
- 新規実装か、既存挙動の修正か

意図が読めない場合は、十分なレビューはできないものとして扱う。

## Trace The Real Execution Path

diff だけで判断しない。変更箇所が実際にどう使われるかを追う。

最低限、次を確認する。

- 変更された関数、型、設定の呼び出し元
- 呼び出し先や依存先
- 入力、分岐、返り値
- 例外、失敗時の挙動
- 状態変更、副作用、永続化
- 外部 API、DB、キュー、ファイル、キャッシュとの境界

変更前と変更後で、どの挙動が変わるかを言語化する。

## Look For Broken Contracts

実装の局所的な正しさだけでなく、契約の破れを探す。

特に次を確認する。

- 関数や API の入出力契約
- nullability、型、デフォルト値
- 公開 API、保存形式、設定値の互換性
- エラーコード、例外、リトライ、タイムアウトの扱い
- ログ、メトリクス、トレースに必要な文脈
- テストが期待する振る舞いとの整合
- migration や rollout / rollback の成立性

## Form Failure Hypotheses Before Writing Findings

指摘を書く前に、壊れそうな点を仮説として洗い出す。

例として、次のような仮説がある。

- 特定の入力で落ちる
- 既存クライアントを壊す
- エラー時に状態が壊れる
- 監視や障害解析ができなくなる
- テストが変更の本質をカバーしていない

仮説が立たない場合でも、主要経路と失敗経路は確認する。

## Start With Findings

返答では、要約より先に findings を並べる。

各 finding には次を含める。

- 重大度
- 問題が起きる条件
- 何が壊れるか
- 根拠となるファイル、行、経路
- 修正の方向性

可能な限り、実害を具体的に書く。
抽象的な懸念だけで強く断定しない。

## Keep The Output Contract Tight

レビュー結果は、次の順で返す。

1. findings
2. 未確認領域 / open questions / assumptions
3. 必要なら短い要約

- 要約を findings より先に出さない
- finding がない場合でも、残リスク、テストギャップ、未確認領域を短く添える
- 確認していない観点を「問題なし」に含めない

## Use The Axes As A Final Coverage Check

最後に、次の観点で見落としを確認する。

- 正確性
- セキュリティ
- パフォーマンス
- 後方互換性
- 可観測性
- 運用性
- テスト容易性
- 可読性
- 一貫性

観点の列挙を先にしない。まず実行経路と契約を見る。

## Do Not Say "No Issues" Lightly

次を確認できていない限り、安易に「問題なし」と言わない。

- 変更意図を把握した
- 主要な実行経路を追った
- 少なくとも 1 つ以上の失敗経路を確認した
- 契約や互換性への影響を見た
- テストが変更の本質をカバーしているか見た

十分に追えていない場合は、
「問題なし」ではなく「未確認のため断定しない」と返す。

## Treat Ambiguity As A Stop Signal

次のケースは曖昧扱いにする。

- 変更の目的や背景が読み取れない
- 実行経路を追うのに必要な周辺文脈が足りない
- 互換性や運用前提の判断材料がない
- テストや利用者影響の範囲が不明

曖昧な場合は、不足情報と未確認領域を明示する。

## Review Before Finishing

終了前に次を確認する。

- findings を要約より先に並べている
- 各 finding に条件、影響、根拠、修正の方向性がある
- finding がない場合も、残リスクか未確認領域を分けている
- 主要経路、失敗経路、契約、テストの確認範囲を説明できる
